✍️ **(1) 과업 정의:**
누가쓰는지 : 회의 후 정리에 시간을 쓰기 어려운 바쁜 실무자

무엇이 나와야 하는지 : 정형화된 회의록

✍️ **(2) 입력 양식:**

(예: 2025-01-15 오후 2시)

[회의 목적]
(예: 2분기 마케팅 전략 수립)

[원본 텍스트]
(날것의 메모 또는 STT 변환 텍스트를 그대로 붙여넣기)

[핵심 포인트]
(예: 예산 결정사항, 일정 변경 내용 등 반드시 포함해야 할 내용)

[톤]
(예: 공식적/격식체, 간결/비즈니스, 친근/구어체 중 선택)

[금지어]
(예: 특정 경쟁사 이름, 내부 은어, 민감한 단어 등 출력에서 제외할 표현)

[특이사항]
(선택) 강조할 내용, 누락 가능성 있는 부분 등

✍️ **(3) 배역(페르소나):**
이름: 김미소 (33세, 제조업 중간관리자)

상황:
- 클라이언트 미팅 후 회의록을 공식 문서로 전달
- 직접 회의록 작성할 시간 없음
- 상위 보고용 요약본 필요

입력 유형: 날것의 회의 메모 + STT 혼합
입력 특징:
- 오탈자 多, 문장 미완성, 화자 구분 없음
- 내용은 많으나 핵심이 흩어져 있음
- 전문 용어 多

목표: 윗선 보고 + 클라이언트 전달에 바로 쓸 수 있는 구조화된 회의록

핵심 니즈:
- 결정사항 + Action Items 정확하게 추출
- 민감한 내부 발언 필터링 (금지어 적용)
- 핵심 포인트 강조 + 미결사항 명확히 분리

기피 요소:
- 너무 긴 요약, 불필요한 미사여구
- 구어체 표현, 추임새, 갈등 표현 노출

톤: 간결 / 공식적 / 격식체

✍️ **(4) 기본 규칙:**
당신은 회의록 전문 작성 AI입니다.
