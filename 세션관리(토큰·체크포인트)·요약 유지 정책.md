# ✨ 세션관리(토큰·체크포인트)·요약 유지 정책

**-AI와 긴 대화를 해도 내용이 사라지지 않게 저장하는 방법**

- **왜 관리가 필요한가?**

```
❌ AI는 대화창을 닫으면 모든 것을 잊어버림
❌ 대화가 길어지면 앞 내용을 스스로 잊기 시작함
❌ 무료 한도 초과시 대화 기록이 날아감
```

> 1️⃣ 세션 체크포인트 만들기   → 중간 저장 포인트 설정
2️⃣ 주기적 요약 보존하기     → 핵심 내용 압축 저장
3️⃣ 컨텍스트 창 관리하기     → 대화 길이 조절 방법
> 

---

## **1️⃣ 세션 체크포인트 만들기 (= 게임의 중간 저장처럼 대화 내용 저장)**

```
**STEP 1. 체크포인트 기록

📌 세션 체크포인트**

저장 일시  : 20XX-XX-XX 오후 XX시
사용 모델  : Gemini Flash
대화 주제  : 블로그 글 작성 자동화

✅ 완료된 작업
   - 프롬프트 초안 작성 완료
   - 블로그 제목 3개 생성 완료

🔄 진행 중인 작업
   - 본문 내용 다듬기 중

⏭️ 다음에 할 작업
   - 마무리 문단 작성
   - 최종 검토

💾 핵심 프롬프트 저장
   "블로그 글 / 주제 : 건강식단 / 500자 / 친근체"
```

```
**Step 2. 체크포인트 기록 타이밍**

[ ] 대화 10회 주고받을 때마다
[ ] 중요한 결과물 나왔을 때
[ ] 자리 비우기 전
[ ] 주제가 바뀔 때
[ ] 무료 한도 70% 도달했을 때
```

## **2️⃣ 주기적 요약 보존하기 (= 긴 대화를 짧게 압축해서 저장)**

```
**STEP 1. AI한테 직접 요약 시키기**

📌 **요약 요청 프롬프트** 

"지금까지 우리가 나눈 대화를 아래 형식으로
 요약해줘.

 - 주제 :
 - 완료한 것 :
 - 결정한 것 :
 - 다음 할 일 :
 - 핵심 키워드 3개 :

 최대한 짧게, 핵심만!"
 -----------------------------------
 
**STEP 2. 요약 보존 파일 만들기**

📁 대화 요약 보존 기록

[요약 #1] 20XX-XX-XX 오후 XX시
주제      : 블로그 자동화
완료한 것 : 프롬프트 3개 작성
결정한 것 : Gemini Flash 메인 모델로 사용
다음 할 일: 테스트 실행
키워드    : 블로그, 자동화, Groq

[요약 #2] 20XX-XX-XX 오후 XX시
주제      : 프롬프트 경량화
완료한 것 : 토큰 74% 절약 방법 정리
결정한 것 : 목록형 프롬프트 사용
다음 할 일: 실전 적용 테스트
키워드    : 경량화, 토큰, 목록형
-----------------------------------

⏰ 언제 요약?
   → 30분 대화마다 1번
   → 대화창 닫기 전 반드시!
```

## **3️⃣ 컨텍스트 창 관리하기 (= AI가 기억할 수 있는 양을 조절하기!)**

## **❓ 컨텍스트 창이 꽉 차면?**

```
**증상 1 : AI가 앞 내용을 모름**
  나  : "아까 말한 블로그 제목 기억해?"
  AI  : "죄송해요, 어떤 제목인가요?" 😰

**증상 2 : 엉뚱한 답변**
  나  : "그거 수정해줘"
  AI  : "무엇을 수정할까요?" 😰

**증상 3 : 오류 발생**
  AI  : "입력이 너무 깁니다" ❌
```

---

## **✅ 해결 방법 = AI 기억은 한계가 있으므로 직접 저장하자.**

```
1️⃣ 대화가 길어지면 → 새 창 열기
2️⃣ 새 창 열 때    → 요약 붙여넣기
3️⃣ 중요 내용은    → 파일에 저장
```
