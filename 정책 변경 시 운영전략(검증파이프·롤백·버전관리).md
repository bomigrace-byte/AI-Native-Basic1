# ✨ 정책 변경 시 운영전략(검증파이프·롤백·버전관리)

**-AI 서비스가 갑자기 바뀌어도 내 작업이 멈추지 않게 준비하는 것**

> 1️⃣ 검증 파이프라인 만들기   → 변경 감지 체크리스트
2️⃣ 롤백 준비하기            → 비상용 백업 플랜
3️⃣ 버전 관리하기            → 모델/프롬프트 기록
> 

## **1️⃣ 검증 파이프라인 만들기 (= AI 정책이 바뀌었는지 확인하는 루틴)**

### 📌 AI 정책 변경 주간 체크리스트

```jsx
**확인 날짜 : ____년 __월 __일**

[ ] Gemini API 정책 변경 여부 확인
    → 접속 : ai.google.dev
    
[ ] Groq 무료 한도 변경 여부 확인
    → 접속 : console.groq.com

[ ] Perplexity 무료 정책 변경 여부 확인
    → 접속 : perplexity.ai/settings

[ ] 내 프롬프트 테스트 결과 정상 여부
    → 테스트 질문 : "안녕, 작동해?"

변경사항 발견 시 → 아래 롤백 플랜 실행!
-----------------------------------

⏰ 언제 확인?
   매주 월요일 5분만 투자!
```

## **2️⃣ 롤백 준비하기 (= 문제 생기면 즉시 대체 모델로 전환)**

```
**STEP 1. 대체 모델 미리 가입해두기**

메인 모델 → 대체 모델 순서로 준비!

┌─────────────────────────────────┐
│  🥇 메인    : Gemini Flash      │
│  🥈 1차 대체 : Groq LLaMA3      │
│  🥉 2차 대체 : perplexity       │
└─────────────────────────────────┘

✅ 가입할 곳
   1. gemini.google.com
   2. console.groq.com
   3. perplexity.ai
   
   
**STEP 2. 비상 연락망 문서 만들기**
       
🚨 AI 서비스 비상 대체 플랜

상황 : Gemini 정책 변경
행동 : Groq 전환

상황 : Groq 한도 초과 or 서비스 중단
행동 : perplexity 전환

상황 : 모든 API 문제 발생
행동 : Claude.ai 웹에서 수동 작업

전환 소요시간 목표 : 5분 이내!  
```

### **3️⃣ 버전 관리하기 (= 프롬프트와 모델 설정을 기록으로 남기기)**

```
**STEP 1. 프롬프트 버전 기록 파일 만들기**
        파일명 : prompt_versions.txt

📁 프롬프트 버전 관리 기록

[v1.0] 20XX-XX-XX
사용 모델  : Gemini Flash
프롬프트   : 블로그 글 작성 / 주제 / 500자
결과 품질  : ⭐⭐⭐⭐ (좋음)
특이사항   : 없음

[v1.1] 20XX-XX-XX
사용 모델  : Gemini Flash
변경 내용  : 말투 조건 추가
프롬프트   : 블로그 글 / 주제 / 500자 / 친근체
결과 품질  : ⭐⭐⭐⭐⭐ (매우 좋음)
특이사항   : 품질 향상 확인

💡 버전 올리는 기준
   - 프롬프트 내용 바꿀 때 → v1.0 → v1.1
   - 모델 자체를 바꿀 때   → v1.0 → v2.0
```

```
STEP 2. 폴더 구조 만들기
        탐색기에서 아래처럼 폴더 생성!

📂 AI_운영관리
 ├── 📄 정책_체크리스트.txt
 ├── 📄 비상_대체플랜.txt
 ├── 📄 prompt_versions.txt
 └── 📂 백업프롬프트
      ├── 📄 블로그용.txt
      ├── 📄 코딩용.txt
      └── 📄 번역용.txt
```

## **📊 완성 후 기대 효과**

```
Before (지금)
❌ AI 정책 바뀌면 작업 중단
❌ 어떤 프롬프트 썼는지 기억 못함
❌ 대체 수단 없어서 당황

After (30분 후)
✅ 정책 변경 즉시 감지
✅ 5분 안에 대체 모델 전환
✅ 프롬프트 히스토리 완벽 보관
```
